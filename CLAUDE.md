# Cursor Skills

This repo contains reusable AI skills for Cursor IDE and Claude Code, plus an agent team system for coordinated workflows.

## Structure

- `skills/` — Cursor IDE skill definitions (SKILL.md files)
- `.claude/rules/` — Always-on coding conventions (fastapi-backend, react-frontend)
- `.claude/skills/` — Claude Code reusable skills (incremental-build, code-review, wiki-generator, frontend, backend, qa)
- `.claude/agents/` — Claude Code agent team definitions (builder, reviewer, documenter, frontend, backend, qa)
- `eval/` — Skill evaluation framework with fixture app and scenarios
- `app/` — Demo FastAPI project (generated by agent team)

## Rules

`.claude/rules/` contains always-on coding convention files loaded into context automatically:

| Rule | Scope |
|------|-------|
| `fastapi-backend` | FastAPI + SQLAlchemy + Pydantic — project structure, async patterns, DI, auth, testing |
| `react-frontend` | React + TypeScript — components, TanStack Query, routing, forms, state, accessibility |

These are not skills or agents — they provide framework-specific best practices as background context for any agent or direct Claude Code usage.

## Skills vs Agents

**Skills** (`.claude/skills/`) contain reusable methodology — how to chunk work, how to review code, how to generate a wiki. They can be used standalone or by any agent.

**Agents** (`.claude/agents/`) are thin wrappers that reference a skill and add team coordination — who to message, how to handle feedback, when to mark tasks complete.

| Agent | Skill it uses | What it adds | Status |
|-------|--------------|--------------|--------|
| builder | incremental-build | Coordinates with lead for approval, sends to reviewer when done | Tested |
| reviewer | code-review | Sends feedback to builder, re-review loop, escalation | Tested |
| documenter | wiki-generator | Reads progress.md and review.md for context, notifies lead | Tested |
| frontend | frontend | Coordinates with backend on API contracts, sends to reviewer | New |
| backend | backend | Coordinates with frontend on API contracts, sends to reviewer | New |
| qa | qa | Reports bugs to devs, generates .qa/report.md, notifies lead | New |

**Status key:** `Tested` = used to build the demo `app/`. `New` = definitions and workflow templates are complete but not yet battle-tested in a real project — may need tuning.

## Agent Team Workflow

This project supports Claude Code agent teams. See `.claude/agents/WORKFLOWS.md` for ready-to-use team prompts.

## Conventions

- Python code uses 4-space indentation
- Shell scripts use 2-space indentation
- Commit messages: imperative mood, explain *why* not *what*
- Generated artifacts go in dotfolders: `.review/`, `.docs/`, `.qa/`
